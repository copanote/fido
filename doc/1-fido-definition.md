# 1. FIDO 정의

## FIDO란?

**FIDO(Fast IDentity Online)**는 비밀번호 없는 인증(Passwordless Authentication)을 실현하기 위해 만들어진 **개방형 인증 표준**이다.

기존 ID/비밀번호 방식은 피싱, 비밀번호 재사용, 서버 데이터 유출 등 구조적 보안 취약점을 안고 있다. FIDO는 이 문제를 비밀번호 자체를 없애는 방향으로 근본적으로 해결하고자 한다.

---

## 핵심 원리: 공개키 암호화

FIDO의 모든 인증은 **공개키 암호화(Public Key Cryptography)** 원리에 기반한다.

### 등록 (Registration)

1. 사용자가 서비스에 처음 FIDO 등록 요청을 보낸다.
2. 사용자 기기(스마트폰 등)에서 **공개키/개인키 쌍**을 생성한다.
3. **공개키**만 서버로 전송하여 저장한다.
4. **개인키**는 기기 내 보안 영역(TEE)에만 저장되며, 절대 외부로 나가지 않는다.

### 인증 (Authentication)

1. 서버가 무작위 챌린지(Challenge) 값을 클라이언트에 보낸다.
2. 사용자가 생체인식(지문, 얼굴) 또는 PIN으로 기기를 해제한다.
3. 기기가 챌린지에 개인키로 **서명(Signature)** 을 만들어 서버로 보낸다.
4. 서버는 저장해둔 공개키로 서명을 검증한다. 검증 성공 시 인증 완료.

```
등록:  [기기] ──공개키 전송──> [서버]
               개인키는 기기 내 보관

인증:  [서버] ──챌린지──> [기기] ──서명──> [서버] ──공개키로 검증──> 성공
```

## 비밀번호 방식 vs FIDO 방식

| 항목 | 비밀번호 방식 | FIDO 방식 |
|------|------------|---------|
| 인증 수단 | 비밀번호 (지식 기반) | 생체인식, PIN, 보안키 (소유·생체 기반) |
| 서버 저장 정보 | 비밀번호 해시값 | 공개키 |
| 피싱 공격 | 취약 (비밀번호 탈취 가능) | 차단 (도메인 바인딩으로 원천 차단) |
| 서버 데이터 유출 시 | 비밀번호 크래킹 위험 | 공개키만 유출 → 무의미 |
| 사용자 경험 | 비밀번호 기억 필요 | 지문/얼굴만으로 간편 인증 |

### 피싱 차단 원리 (도메인 바인딩)

FIDO는 키 쌍을 등록할 때 **서비스의 도메인/앱 ID와 묶어서** 저장한다.
예를 들어 `mybank.com`에 등록한 키는 피싱 사이트 `myb4nk.com`에서는 절대 작동하지 않는다.
사용자가 피싱 링크를 눌러도 인증 자체가 거부된다.

## FIDO Alliance

FIDO 표준은 **FIDO Alliance**라는 비영리 컨소시엄이 제정하고 관리한다.

- 2012년 설립
- 삼성전자, 구글, 마이크로소프트, 애플, 페이팔, 비자, 마스터카드, 인텔 등 수백 개 기업 참여
- 표준 제정, 인증(Certification) 프로그램 운영, 상호 운용성(Interoperability) 보장

### 인증 프로그램의 의미

FIDO Alliance는 기기·서버 제품이 스펙을 올바르게 구현했는지 심사하는 공식 인증 프로그램을 운영한다.
인증을 통과한 제품에만 "FIDO Certified" 마크를 부여한다.
우리 FIDO 서버도 이 인증 취득 여부가 고객사 영업 시 신뢰성 지표가 된다.

---

## 요약

> FIDO는 "비밀번호 대신, 내 기기 안에 있는 개인키로 인증한다"는 개념이다.
> 개인키는 기기 밖으로 나가지 않고, 서버에는 공개키만 있으므로
> 서버가 해킹되거나 피싱을 당해도 인증 정보가 도용될 수 없다.
