# 3. FIDO 연혁 / 히스토리 / 현상황

---

## 연대기

### 2012년 — FIDO Alliance 설립

- PayPal, Lenovo 등이 주도하여 FIDO Alliance 비영리 컨소시엄 설립
- 배경: 비밀번호 기반 인증의 보안 취약성이 산업 전반의 문제로 대두
- "강력한 인증을 더 쉽게(Simpler, Stronger Authentication)"를 슬로건으로 시작

### 2013년 — 회원사 확대

- 구글, 마스터카드, NXP, 블랙베리, 삼성전자 등 주요 기업 합류
- 스펙 초안 공개 및 업계 피드백 수렴

### 2014년 — FIDO 1.0 (UAF + U2F) 공식 발표

- **UAF 1.0**: 모바일 생체인증 표준 발표
- **U2F 1.0**: 하드웨어 보안키 기반 2FA 표준 발표
- 삼성 갤럭시 S5의 지문인식 로그인이 UAF 기반으로 구현됨 → **최초 대규모 상용화**
- PayPal이 UAF 기반 지문 결제 서비스 출시 (갤럭시 S5 대상)

### 2015년 ~ 2016년 — 모바일 결제 확산

- 삼성페이(2015년 출시)에 FIDO UAF 적용
- 애플페이는 자체 보안 아키텍처 사용하나 유사한 원리
- 국내 은행 앱들이 지문인증 도입 시작 (우리은행, 신한은행 등)
- 구글이 안드로이드에 FIDO 클라이언트 내장 시작

### 2017년 — UAF 1.1 발표

- 트랜잭션 확인(Transaction Confirmation) 기능 강화
  - 예: "50,000원을 A계좌로 이체하겠습니까?" 메시지를 사용자가 직접 확인하고 지문 인증
- 금융 거래 보안 강화에 적합한 기능 추가

### 2018년 — FIDO2 발표 (WebAuthn + CTAP2)

- W3C와 FIDO Alliance 공동으로 **WebAuthn** 표준 완성
- 크롬 67, 파이어폭스 60에 WebAuthn 기본 탑재
- 비밀번호 없는 **웹 인증** 시대 개막
- CTAP2 발표로 외부 인증기(USB 키, 스마트폰)와 브라우저 간 통신 표준화

### 2019년 — WebAuthn W3C 공식 권고안 승인

- W3C가 WebAuthn을 공식 Recommendation(권고안)으로 채택
- 마이크로소프트 Windows Hello, 엣지 브라우저에서 FIDO2 완전 지원
- iOS 13 / macOS Catalina에서 Safari WebAuthn 지원 추가

### 2020년 — 국내 공인인증서 의무 사용 폐지

- 전자서명법 개정으로 공인인증서 독점 지위 폐지 (2020년 12월)
- FIDO 기반 간편인증이 공인인증서의 유력한 대안으로 급부상
- 카카오, 네이버, 통신사 등이 FIDO 기반 간편인증 서비스 출시

### 2021년 — 국내 금융권 FIDO 전면 도입

- 대부분의 국내 시중은행·인터넷은행 앱이 FIDO UAF 기반 생체인증 표준으로 채택
- 금융보안원이 FIDO 기반 인증 가이드라인 발표

### 2022년 — Passkey 시대 개막 (업계 최대 분기점)

- 애플(WWDC 2022), 구글, 마이크로소프트가 **동시에** Passkey 지원 공동 선언
- iOS 16, macOS Ventura에서 Passkey 정식 지원 (iCloud 키체인 동기화)
- Android 9+, Chrome에서 Passkey 지원 (구글 비밀번호 관리자 동기화)
- Passkey의 핵심: 기기 간 동기화로 기기 분실 문제 해결 + 크로스 디바이스 인증

### 2023년 — Passkey 본격 보급

- 구글 계정: 모든 사용자에게 Passkey 기본 옵션 제공
- 애플 ID, 마이크로소프트 계정도 Passkey 지원
- GitHub, Shopify, PayPal, KAYAK 등 주요 웹 서비스가 Passkey 도입
- 국내: 네이버, 카카오 등 Passkey 도입 검토 시작

### 2024년 ~ 현재 — Passkey 주류화

- FIDO Alliance가 "2024년은 Passkey의 해"로 선언
- 국내 주요 금융사·포털의 Passkey 도입 가속화
- FIDO2가 사실상의 **차세대 글로벌 인증 표준**으로 자리잡음

## 우리 회사 위치

### 현재

- 삼성페이 연동을 위한 **FIDO UAF 서버** 구축·운영 중
- UAF 1.x 기반 모바일 생체인증 서비스 제공 가능 상태

### FIDO 시장에서의 포지션

```
FIDO 표준 진화 방향:
UAF (2014) → FIDO2/WebAuthn (2018) → Passkey (2022~현재)

우리 현재 위치:
[UAF] ←── 여기 있음
                [FIDO2] [Passkey] ←── 시장이 이쪽으로 이동 중
```

### 고객사 영업을 위한 시사점

1. **UAF 서비스는 여전히 유효**: 국내 모바일 앱 기반 금융 서비스는 여전히 UAF가 주류
2. **FIDO2 수요 증가**: 웹 서비스나 글로벌 대응이 필요한 고객사는 WebAuthn 요구
3. **Passkey는 미래 방향**: 중장기적으로 Passkey 지원 여부가 경쟁력이 될 것
4. **인증(Certification) 취득**: FIDO Alliance 공식 인증이 있으면 고객사 신뢰도 높일 수 있음
