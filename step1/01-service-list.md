# 현재 UAF 서버로 제공 가능한 FIDO 서비스 목록

현재 보유한 UAF 1.1 서버를 기반으로 고객사에 제공할 수 있는 서비스 목록.

---

## 전제 조건 (현재 우리가 가진 것)

```
보유 자산:
  ✅ UAF 1.1 서버 (삼성페이 연동으로 검증됨)
     - 등록(Registration) API
     - 인증(Authentication) API
     - 해지(Deregistration) API
     - 공개키 저장 DB
     - 챌린지 생성/검증 로직
     - Attestation 검증

  ❓ App SDK (삼성페이 전용인지, 범용인지 확인 필요)
  ❓ FIDO Alliance 공식 인증 보유 여부 확인 필요
```

---

## 제공 가능한 서비스 목록

### ✅ 즉시 제공 가능 (서버 재활용)

#### 1. 모바일 앱 생체 로그인

```
서비스 내용:
  고객사 앱에 지문/Face ID 기반 로그인 기능 제공

대상 고객사:
  - 은행/증권 앱
  - 쇼핑몰 앱
  - 기업 내부 업무 앱
  - 헬스케어 앱

동작:
  고객사 앱 → (우리 SDK) → 우리 UAF 서버 → 인증 결과 반환

수익 모델:
  - 서버 API 호출 건당 과금
  - 월정액 SaaS 형태
  - 온프레미스 서버 납품
```

#### 2. 모바일 앱 생체 본인인증

```
서비스 내용:
  회원가입, 계좌 개설 등 본인 확인이 필요한 시점에 생체인증

기존 본인인증(SMS, 아이핀) 대비 장점:
  - SIM 스와핑 공격 불가
  - 사용자 경험 향상 (문자 수신 불필요)
  - 해외에서도 사용 가능

대상 고객사:
  - 핀테크 (계좌 개설)
  - 보험사 (계약 체결)
  - 공공기관 (민원 신청)
```

#### 3. 모바일 간편 결제 인증

```
서비스 내용:
  결제 시 생체인증으로 승인 (삼성페이와 동일한 구조)
  UAF 1.1 Transaction Confirmation 기능 활용:
  "○○마트에서 50,000원 결제" 문구를 사용자가 확인 후 지문 인증

대상 고객사:
  - 간편결제 사업자
  - 카드사 앱
  - 전자상거래 플랫폼
```

#### 4. 모바일 전자서명 / 문서 서명

```
서비스 내용:
  계약서, 동의서 등에 생체인증 기반 전자서명
  UAF Transaction Confirmation으로 서명 대상 문서 내용 표시 후 서명

대상 고객사:
  - 보험사 (청약서 서명)
  - 부동산 플랫폼 (계약서)
  - HR 시스템 (근로계약서)
  - 의료기관 (동의서)
```

#### 5. 기업 내부 시스템 접근 인증 (모바일)

```
서비스 내용:
  임직원이 스마트폰 생체인증으로 사내 시스템 접근
  VPN 없이 안전한 Zero Trust 인증

대상 고객사:
  - 중견/대기업 IT 부서
  - 금융사 내부 보안팀
```

---

### ⚠️ 일부 추가 개발 후 제공 가능

#### 6. FIDO2 / WebAuthn 서버 (브라우저 인증)

```
서비스 내용:
  PC/모바일 웹 브라우저에서 비밀번호 없는 로그인
  Passkey 지원

현재 상태: UAF 서버만 있음 → FIDO2 서버 추가 개발 필요
개발 규모: 중간 (UAF 서버 구조 재활용 가능, 프로토콜 레이어 교체)

대상 고객사:
  - 웹 서비스 중심 고객사
  - 글로벌 대응 필요 고객사
```

#### 7. FIDO + OTP 하이브리드 인증

```
서비스 내용:
  FIDO 1차 인증 + OTP 2차 인증 조합
  규제상 다중 인증(MFA)이 필수인 고객사 대상

현재 상태: OTP 서버 연동 추가 필요
```

## 확인이 필요한 사항

현재 UAF 서버 상태에 대해 아래를 확인해야 정확한 서비스 범위를 결정할 수 있습니다.

| 확인 항목 | 중요도 | 이유 |
|----------|--------|------|
| App SDK가 삼성페이 전용인지 범용인지 | 🔴 높음 | SDK 재사용 가능 여부 결정 |
| FIDO Alliance 공식 서버 인증 여부 | 🔴 높음 | 고객사 신뢰도, B2B 영업 필수 |
| 멀티 테넌트 구조 지원 여부 | 🔴 높음 | 여러 고객사 동시 서비스 가능 여부 |
| UAF 버전 (1.0 vs 1.1) | 🟡 중간 | Transaction Confirmation 사용 가능 여부 |
| 현재 SLA / 서버 가용성 수준 | 🟡 중간 | 금융권 고객사 요구 수준 충족 여부 |
| 온프레미스 납품 가능 여부 | 🟡 중간 | 보안 규정상 외부 서버 불가한 고객사 대응 |
