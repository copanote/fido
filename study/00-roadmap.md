# FIDO 학습 로드맵

이 폴더는 FIDO 인증 기술을 단계적으로 이해하기 위한 학습 자료 모음입니다.
순서대로 읽으면 "공개키 암호화 기초"부터 "실제 서버 구현"까지 전체 그림을 잡을 수 있습니다.

---

## 학습 순서

```
[1단계: 기초]
  01-crypto-basics.md       ← 공개키 암호화, 디지털 서명 이해
                              (FIDO의 뿌리가 되는 암호학 개념)

[2단계: UAF 이해]
  02-uaf-deep-dive.md       ← UAF 프로토콜 상세 분석
                              (우리가 구축한 스펙. 시퀀스 다이어그램 포함)

[3단계: FIDO2 이해]
  03-fido2-webauthn.md      ← WebAuthn + CTAP 상세 분석
                              (브라우저 표준. 코드 예시 포함)

[4단계: 서버 구조]
  04-server-architecture.md ← FIDO 서버 내부 구조와 설계
                              (DB 스키마, API 설계, 검증 로직)

[5단계: Passkey]
  05-passkey.md             ← Passkey 심층 분석
                              (크로스 디바이스 인증, 동기화 원리)

[6단계: 실전]
  06-attack-and-defense.md  ← 보안 위협과 방어 전략
                              (FIDO가 막는 공격과 남아있는 위협)
```

## 권장 학습 방법

1. 01번부터 순서대로 읽는다.
2. 시퀀스 다이어그램을 직접 손으로 따라 그려본다.
3. 코드 예시는 실제로 동작하는 코드가 아닌 **개념 설명용 의사코드(pseudocode)** 이므로, 흐름을 이해하는 데 집중한다.
4. 모르는 용어가 나오면 먼저 01번으로 돌아간다.

## 빠른 용어 사전

| 용어 | 한 줄 설명 |
|------|----------|
| 공개키 (Public Key) | 누구에게나 알려줘도 되는 키. 서버에 저장. |
| 개인키 (Private Key) | 절대 공개하면 안 되는 키. 기기 내 보안 영역에 저장. |
| 챌린지 (Challenge) | 서버가 매번 새로 만드는 무작위 값. 리플레이 공격 방지. |
| 서명 (Signature) | 개인키로 데이터를 암호화한 값. 개인키 소유자만 만들 수 있음. |
| 검증 (Verification) | 공개키로 서명이 유효한지 확인하는 행위. |
| TEE | 스마트폰 내 보안 실행 환경. 일반 앱이 접근 불가. |
| RP (Relying Party) | FIDO를 도입하는 서비스 제공자 (은행, 쇼핑몰 등). |
| Authenticator | 실제 인증을 수행하는 하드웨어/소프트웨어 (지문 센서 등). |
| Attestation | 인증기가 FIDO 공식 인증을 받은 정품임을 증명하는 것. |
| Assertion | 인증 시 인증기가 서버에 보내는 서명된 응답. |
